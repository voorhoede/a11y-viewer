The A11y Viewer

Use this A11y Viewer during the development of your project to check for some of the most basic accessibility issues. With the A11y Viewer you can emulate blindness, color-blindness and even disable clicks and so you can only use keys to navigate.

Motivation

We wanted to simulate blindness, color blindness and disability to use mouse.

Examples

Below are some examples of (not so) accessible websites. These are just a few examples that we could find. The good ones make good use of contrast and are easy to navigate with the TAB-key. The other ones, not so much.

Any additions to these lists are welcome.

Good:

  • Nomensa - Good contrast
  • 24 Ways - Good navigation
  • CSS Zen Garden - Good contrast

Bad:

  • Airtable - Low contrast
  • Brit + Co - Bad navigation
  • Sears - Bad navigation

Limitations

  • No websites that use the HTTPS protocol are supported yet.
  • Website which have headers with 
    X-Frame-Options
    set to 
    SAMEORIGIN
    .
  • Website which have headers with 
    X-Frame-Options
    set to 
    deny
    .
  • The following browsers are not supported yet: IE8 - IE11.
